# 입찰 공고 분석기 PRD

## 1. 배경
선엔지니어링 총무팀은 나라장터 입찰 공고문을 검토할 때마다 마감일·참가자격·제출서류를 일일이 찾아 읽고, 여러 공고를 동시에 비교하기 어려우며, 제출서류(유사용역 실적·기술자 경력)를 매번 새로 작성한다. 이 반복 작업의 소요 시간을 줄인다.

## 2. 목표
공고 PDF를 업로드하면 핵심 정보가 카드로 정리되고, 마감일 임박도(D-Day)와 제출서류 준비 상태를 한눈에 관리하며, 여러 공고를 나란히 비교하고, 반복 작성하는 실적/경력 서류를 폼으로 관리해 문서로 내보낼 수 있게 한다.

## 3. 범위 (이번 버전)
**포함**
- PDF 텍스트 추출 + 키워드/정규식 기반 필드 매칭 (공고명, 발주기관, 예정금액, 용역기간, 마감일, 참가자격, 제출서류)
- D-Day 계산 및 임박(7일 이내) 강조
- 제출서류 체크리스트 및 준비 진행률(%)
- 여러 공고 나란히 비교하는 뷰
- 유사용역 실적·기술자 경력 폼 작성 및 문서 파일로 내보내기
- 공고/체크리스트 데이터는 Supabase(공유 데이터베이스)에 저장해 여러 기기·사용자가 동일하게 조회. 실적·경력 정보는 브라우저 localStorage에 보관 (로그인 없음)

**제외 (향후 확장 후보)**
- 실제 AI 기반 문서 이해/분석
- 로그인·권한 관리 (현재는 인증 없이 누구나 공고 데이터를 읽고 쓸 수 있음)
- 스캔 이미지 PDF의 OCR 처리
- 진짜 바이너리 .hwp 자동 생성 (별도 검토 후 확정)

## 4. 핵심 기능

### 4.1 공고 PDF 업로드 → 카드 정리
사용자가 텍스트 기반 PDF 공고문을 업로드하면, 브라우저에서 텍스트를 추출하고 키워드/정규식으로 다음 항목을 뽑아 카드에 표시한다: 공고명, 발주기관, 예정금액, 용역기간, 마감일, 참가자격, 제출서류.
- 실제 AI 분석이나 서버 전송 없이 규칙 기반 추출만 사용한다.
- 항목별로 추출 성공 여부를 표시하고, 실패한 항목은 바로 수동 입력할 수 있어야 한다.
- 카드에는 **"⚠ 원문 확인 필요"** 경고를 상시 노출한다 (추출 정확도를 보장하지 않음을 항상 알림).
- 스캔 이미지 PDF 등 텍스트 추출이 전혀 안 되는 경우, 실패를 알리고 모든 필드를 수동 입력하도록 안내한다.

### 4.2 D-Day 표시
마감일 기준으로 오늘까지 남은 일수를 계산해 카드/상세/비교 화면에 "D-n"으로 표시한다. 7일 이내로 남았으면 빨간색으로 강조해 시각적으로 눈에 띄게 한다. 마감일 추출에 실패한 공고는 D-Day 대신 "마감일 미확인" 상태를 보여준다.

### 4.3 제출서류 체크리스트 & 진행률
추출된 제출서류 목록을 체크리스트로 만들고, 사용자가 항목을 완료 처리하면 체크 상태가 저장된다. 체크된 항목 비율을 진행률(%)로 계산해 카드와 상세 화면에 표시한다. 체크리스트 항목은 수동으로 추가/삭제할 수 있다.

### 4.4 공고 비교
등록된 공고가 2개 이상이면 비교 화면에서 선택한 공고들을 나란히 놓고 예정금액·마감일(D-Day)·참가자격·제출서류 준비 진행률을 비교할 수 있다.

### 4.5 실적·경력 폼 & 문서 내보내기
유사용역 수행 실적(발주처, 계약금액, 계약기간, 개요 등)과 투입 기술자 경력을 폼으로 여러 건 입력·관리할 수 있다. 입력한 내용을 문서 파일로 내보내 나라장터 제출용 서류 작성 시간을 줄인다. (내보내기 파일 형식은 "5. 문서 내보내기" 참고)

## 5. 데이터 보관
공고 카드와 체크리스트 상태는 **Supabase(PostgreSQL) 데이터베이스**에 저장한다. 로그인이 없는 내부 도구 특성상 인증 없이 anon 키로 읽고 쓰며, 새로고침하거나 다른 기기·브라우저에서 접속해도 동일한 데이터가 보인다.
실적·경력 데이터는 여전히 브라우저 **localStorage**에 보관한다 (기기/브라우저 종속, 캐시 삭제 시 소실). 이 차이를 화면 안내 문구로 명시한다.

## 6. 문서 내보내기 형식
바이너리 .hwp나 한글 전용 .hwpx를 처음부터 직접 생성하는 것은 신뢰할 만한 오픈소스 라이브러리가 없어 파일 손상 위험이 크다. 1차 버전은 **.docx**로 내보내며, 최신 한글(HWP) 프로그램은 .docx를 직접 열고 편집한 뒤 "다른 이름으로 저장"으로 .hwp 변환이 가능하다는 점을 안내 문구로 함께 제공한다. 실제 .hwp/.hwpx 직접 생성 여부는 별도로 검토해 추후 결정하며, 내보내기 구현은 형식 교체가 쉽도록 하나의 모듈로 격리한다.

## 7. 사용성 원칙
- 로그인/회원가입 없음. 접속 즉시 사용 가능.
- 비개발자 기준으로 용어와 화면 흐름을 단순하게 유지 (업로드 → 확인/수정 → 완료).
- 자동 추출 결과는 항상 사람이 원문과 대조해 확인해야 함을 지속적으로 안내한다.

## 8. 성공 기준
- 공고 PDF 업로드부터 핵심 정보 확인까지 걸리는 시간이 수작업 대비 크게 단축된다.
- 여러 공고의 마감일·조건을 한 화면에서 비교할 수 있다.
- 실적/경력 서류를 매번 새로 작성하지 않고 재사용/내보내기로 대체할 수 있다.

## 9. 향후 확장 (비범위)
- 실제 AI/LLM 기반 공고 이해 및 요약
- 서버 저장 및 팀 단위 공유, 로그인
- 스캔 PDF OCR
- 진짜 .hwp/.hwpx 네이티브 생성
